# Learning Route Map

The project aims to improve AI product manager competitiveness through outputs, not passive study.

## Capability Tracks

- AI Evals / Harness / LLMOps: eval datasets, experiment runs, traces, production scoring, CI gates, failure attribution, feedback loops.
- AI Builder: AI-assisted development, Python/FastAPI basics, simple frontend prototypes, LLM API engineering, RAG/Agent runnable demos, tests, evals, and technical retrospectives.
- LLM platform product: model access, routing, fallback, usage, cost, quotas, pricing, permissions, monitoring.
- RAG/Agent quality governance: RAG evaluation, Agent Harness, tool registry, state, guardrails, trace, regression testing, recovery.
- Solution/FDE delivery: keep only as a supporting capability for customer problem discovery and bad-case collection.
- Industry AI scenario: manufacturing, automotive, enterprise services, government-enterprise digitization.
- Career expression: resume bullets, case studies, interview scripts, portfolio demos.

## Default Sequence

1. Learn enough to build or explain a concrete artifact.
2. Build or refine the artifact with AI-assisted development when useful.
3. Verify the artifact with tests, evals, traces, or metrics.
4. Convert it into product reasoning.
5. Convert reasoning into resume/interview material.
6. Evaluate whether it improves target-role competitiveness.

## Resource Priority Sequence

1. Chinese foundation: Datawhale LLM Cookbook and Happy-LLM application chapters.
2. AI Builder basics: Datawhale LLM Universe, FastAPI 中文文档, and this project's LLM Gateway codebase.
3. Eval Harness practice: promptfoo minimal evals and CI gate concepts.
4. Selected AI Builder practice: rohitg00/ai-engineering-from-scratch Phase 11/13/14/17 lessons that map to LLM Gateway, Eval Harness, RAG Evaluation Harness, Agent Harness, or LLMOps.
5. Agent Harness learning: Hugging Face Agents Course 中文版, limited to Agent basics, LangGraph, Agentic RAG, observability, and evaluation.
6. Product teardown: Langfuse, OpenAI Evals, and promptfoo through product capability, information architecture, metrics, permissions, and reporting.
7. Tool comparison: Phoenix, DeepEval, RAGAS only when a concrete project question appears.

Do not schedule broad course catalogs, platform homepages, full long English engineering repositories, model-training courses, algorithm drills, or complex full-stack tracks as continuous study.

## Task Sizing

- Weekday morning: build, analyze, or write a core artifact.
- Weekday evening: update resume, interview script, or notes; avoid heavy coding.
- Saturday morning: concentrated project improvement.
- Sunday morning: review, scoring, next-week planning.
