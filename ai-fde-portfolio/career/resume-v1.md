## 项目经历：大模型调用治理平台

项目定位：

面向企业客户和内部平台运营的大模型调用治理平台，解决多模型接入后的路由、稳定性、成本、日志和评测问题，并通过 Eval Harness 支持可重复测试和失败归因。

可写进简历的 bullet：

- 设计并实现多模型统一调用入口，支持按默认、低价优先、最快优先、综合策略进行模型路由，降低客户接入多供应商模型的复杂度。
- 建立调用日志和成本统计机制，记录应用、模型、供应商、Token、费用、耗时、状态和 fallback 链路，为客户用量分析和内部毛利控制提供数据基础。
- 设计应用级预算控制和 QPS 限流策略，支持超预算后降级到低价模型或拦截请求，提升平台商业化治理能力。
- 引入 Prompt 版本管理和 Eval Harness，将模型调用从“能用”推进到“可重复评测、可追踪、可优化、可复盘”。
- 设计批量评测 case 结构，输出通过率、失败率、平均成本、平均耗时、fallback 次数和失败归因，辅助定位模型、路由、Prompt 或策略问题。
- 梳理平台从普通 API 中转升级为 LLMOps 治理平台的产品路径，覆盖稳定性、成本、权限、质量和客户套餐差异化。
