# 6 个月 AI/FDE 转型路线

## 定位

目标岗位优先级：

1. AI 解决方案 / FDE 型产品
2. 大模型平台产品
3. 技术型 AI 产品经理
4. AI 应用开发型产品，作为备选加分方向

你的卖点组合：

- 软件工程 + Java 开发经验
- 产品经理经历
- 大模型平台/中转平台经验
- 外包、驻场、客户现场对接经验
- 重庆本地制造业、政企数字化、企业服务场景适配度

## 第 1 个月：LLM Gateway / LLMOps Mini Platform + Eval Harness V0

目标：把现有“大模型中转平台经验”升级成“调用治理平台”项目。

必须完成：

- 多模型接入
- 路由策略
- fallback
- 调用日志
- Token 与成本统计
- 应用级用量报表
- 预算和限流
- Prompt 版本
- Eval Harness V0：测试集、批量运行、trace、成本/耗时/fallback 指标、失败归因
- 简历 V1
- 项目复盘 V1

## 第 2 个月：RAG Evaluation Harness & Knowledge Governance

目标：不做普通知识库，做企业上线后真正需要的质量闭环。

必须覆盖：

- 标准问答集
- 批量评测
- 召回命中分析
- Bad Case 管理
- 知识缺口识别
- 文档版本回归测试
- 引用可信度与幻觉风险
- 不同切分、Embedding、重排策略的 A/B 对比

## 第 3 个月：AI 解决方案/FDE 助手 + Agent Harness Layer

目标：把客户现场经验转成 AI 方案生成和交付闭环。

必须覆盖：

- 客户需求抽取
- 行业、痛点、预算、约束识别
- 方案模块匹配
- 实施计划生成
- 风险清单
- 待确认问题
- 人工确认节点
- 修改历史
- Agent loop、Tool registry、State、Guardrails、Trace、Recovery

## 第 4 个月：行业场景项目

默认方向：制造业售后诊断 Agent。

原因：更贴近重庆就业市场里的汽车、制造、装备、政企数字化项目。

必须覆盖：

- 故障描述解析
- 维修手册 RAG
- 多轮追问
- 诊断路径
- 工单生成
- 人工升级
- 工具调用轨迹和失败恢复

备选方向：企业经营数据分析 Copilot。

## 第 5 个月：作品集与面试启动

目标：把项目变成市场报价。

必须完成：

- 简历 V2
- 3 个项目作品集
- 每个项目 3 分钟、8 分钟、15 分钟讲法
- 每周 5-10 个岗位投递
- 面试反馈复盘

## 第 6 个月：决策与谈判

判断标准：

- 本公司涨到 11K-12K 且给 AI 产品/方案负责人机会：可继续观察。
- 本公司只涨 500-1000 或继续模糊承诺：继续找。
- 外部 13K-15K 且方向匹配：认真考虑。
- 外部低于 11K：除非平台极强，否则不建议动。
