# 项目设计：Agent Harness Layer

## 项目定位

面向 AI Evals / Harness / LLMOps 质量治理岗位，展示 Agent 从“能跑”到“可追踪、可评测、可恢复、可人工接管”的产品化能力。

客户现场和 FDE 经验只作为问题来源：把现场需求不清、系统复杂、交付压力、失败难排查等问题转化为 Agent Harness 的 trace、状态、权限、失败归因和回归测试能力。

## 输入

- 客户会议纪要
- 需求描述
- 招标文件片段
- 现有系统说明
- 预算和上线周期约束

## 输出

- Agent 任务目标和边界
- 工具注册与权限
- 状态 schema
- 运行 trace
- 失败归因
- 人工确认节点
- 回放与回归测试结果
- Bad Case 转评测集规则

## Agent Harness Layer

- Agent loop：计划、工具调用、观察、下一步决策、终止条件。
- Tool registry：可用工具、参数 schema、权限边界。
- State：客户信息、约束、已确认问题、方案版本。
- Guardrails：敏感信息、预算、权限、人工确认节点。
- Trace：每一步为什么调用工具、工具返回了什么、是否失败。
- Recovery：工具失败、信息不足、循环调用、超时后的处理策略。
- Regression：工具、Prompt、模型或工作流变更后的回归评测。

## 面试价值

这个项目能体现你不是只会做后台管理系统，也不是只会调模型，而是理解企业 Agent 的质量治理。重点不是包装成 FDE，而是说明你能把客户现场里的模糊需求、失败案例和生产反馈转成可评测、可追踪、可回归、可恢复的 Agent Harness。
