# 项目设计：RAG Evaluation Harness & Knowledge Governance

## 项目定位

不做普通“上传文档问答”的知识库，而是做企业 RAG 上线前后的 Evaluation Harness 和质量闭环。

## 要解决的问题

- 用户问不到答案时，是召回问题、知识缺失，还是模型生成问题？
- 文档更新后，原来的高频问题是否还能答对？
- 引用来源是否可靠？
- 哪些问题反复失败，应该补知识还是改切分策略？
- 如何把 Bad Case 变成可持续优化的产品流程？

## 核心模块

- 文档版本管理
- 标准问答集
- 批量评测
- 召回命中分析
- Bad Case 管理
- 知识缺口识别
- 回归测试
- 幻觉风险评分
- 切分策略 / Embedding / 重排策略 A/B 对比
- 引用来源覆盖率
- 失败归因：召回失败、生成失败、知识缺失、引用缺失、权限问题

## 面试讲法

很多知识库 Demo 的问题是演示时能答，但上线后没人知道答错在哪里。我会把 RAG 项目重点放在 Evaluation Harness 和治理，而不是单次问答。这样才能体现生产可用性：每次文档更新、Prompt 修改、切分策略调整，都能用同一批 case 做回归测试，知道质量有没有变好。
